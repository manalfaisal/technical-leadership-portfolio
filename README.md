# Technical Leadership Portfolio — Manal Faisal

## About Me / Leadership Arc
When code frustrates me, my default communication pattern is trying to figure it out on my own before asking for help. I usually spend a lot of time troubleshooting independently because I like understanding where something went wrong, but sometimes that means I wait too long to reach out or end up procrastinating when I'm stuck.

The stakeholder I'm least likely to consider in my technical work is someone who is completely new to coding. Once I've learned something, it's easy to forget which parts were confusing when I first started and accidentally assume more background knowledge than someone might have.

If I was going to pick one technical community to make better this summer, it would be the KWK scholar community because I've seen how much of a difference clear communication, encouragement, and support can make when someone is first learning to code. I want technical learning spaces to feel more collaborative and less intimidating for people who are still building confidence in their skills.
